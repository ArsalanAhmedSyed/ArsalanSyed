Final Year Project

Project:

3rd Year, Independent, Unreal Engine, C++


Learning Outome:

  • Gameplay Programming
  • Designer Consideration
  • Design Patterns
  • UML Diagram

Gameplay Features:

  • Abilities (Single & Combo casting)
  • Level Up (Player & Abilities)
  • AI Monsters (Drop Item)
  • Vendor
  • Inventory

This project was my first step towards becoming a gameplay programmer and expanding my Unreal C++ knowledge. It is hard to believe that in 5 months I have managed to learn Unreal C++ and expand my Unreal Engine knowledge, as at the start of it I had no to very little knowledge regarding the gameplay programmer role and Unreal C++. I only had knowledge of Blueprint and creating prototypes in Unreal Engine.

Furthermore, I focused on learning UML diagrams and design patterns to improve my programming skills and create flexible or maintainable project. In addition to that, I focused on using data assets and providing tool tips for all the features that were accessible in the editor or blueprint as if I were making them for a designer.

I have also taken inspiration from the GAS plugin to create my own mini-ability system, as the project contains attribute setup, an ability component, and a damage system (based on UGameplayEffect).

Overall, the project goal was to create mini-RPG-style gameplay. Focusing on creating flexible code structure as well as easy setup allowing designers to create new items or gears, as well as easy to tweak properties (in this case using Data Asset).
